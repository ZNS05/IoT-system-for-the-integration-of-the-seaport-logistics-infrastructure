# IoT-система для интеграции логистической инфраструктуры морского порта

## Описание проекта

Данный проект представляет собой учебное IoT-решение, разработанное в рамках дисциплины
**«Многоуровневая организация программных систем (МОПС)»**.

Цель проекта — **повышение операционной эффективности портового терминала** за счёт внедрения IoT-системы для сбора, передачи и последующей обработки телеметрических данных от оборудования порта (краны, погрузчики, транспорт).

---

## Цель и задачи

### Цель

Повышение эффективности логистических процессов морского порта на основе IoT-подходов.

### Задачи

- приём телеметрических данных от устройств;

- валидация и сохранение данных;

- асинхронная передача данных для последующей обработки;

- обеспечение масштабируемой и расширяемой архитектуры.


---

## Архитектура системы

На текущем этапе реализованы следующие компоненты:

### IoT Controller

HTTP-сервис (FastAPI), выполняющий:

- приём телеметрии (`POST /ingest`);

- валидацию входящих данных;

- сохранение данных в MongoDB;

- публикацию сообщений в очередь RabbitMQ.


### MongoDB

Документо-ориентированная база данных, используемая для:

- хранения телеметрических данных (`measurements`);

- обеспечения отказоустойчивости (данные сохраняются независимо от обработки).


### RabbitMQ

Очередь сообщений, используемая для:

- асинхронной передачи телеметрии;

- развязки IoT Controller и последующих обработчиков данных;

- буферизации сообщений.

					(ожидает обработки)`

## Запуск проекта

### 1. Запуск инфраструктуры

`docker compose up -d`

Будут подняты:

- MongoDB

- RabbitMQ (с Web UI)


### 2. Запуск IoT Controller

`uvicorn main:app --reload --host 0.0.0.0 --port 8000`

### 3. Проверка

- Health check:
    `GET http://localhost:8000/health`

- Приём телеметрии:
    `POST http://localhost:8000/ingest`
